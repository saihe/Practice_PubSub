# 画面仕様書

対象画面は **1 画面のみ**：タスク管理（一覧）画面。

## 1. 画面の目的
- タスクをキューに追加（publish）し、購読側ワーカによる非同期処理の進捗・結果を一覧で確認する。
- 実行中タスクを中止できる。
- 自動更新（ポーリング）で最新状態を表示し、当セッションで投入したタスクの終了をスナックバーで知らせる。

## 2. レイアウト（ゾーン構成）

```
┌────────────────────────────────────────────────────────────────┐
│ (A) ヘッダー   タイトル「Pub/Sub タスク管理」＋ 説明文            │
├────────────────────────────────────────────────────────────────┤
│ (B) エラーバナー（任意・通信失敗時のみ表示）                      │
├────────────────────────────────────────────────────────────────┤
│ (C) ツールバー                                                   │
│   [タスク実行（追加）] [画面更新] [タスク中止]   自動更新:Ns/最終更新  [間隔セレクト▼] │
├────────────────────────────────────────────────────────────────┤
│ (D) タスク一覧テーブル                                            │
│   ○ | タスクID | 追加日時 | ステータス | 実行時間 | 処理結果 | 処理件数 | 全体件数 │
│   …行…                                                           │
├────────────────────────────────────────────────────────────────┤
│ (E) フッター注記（中止の操作説明）                                │
└────────────────────────────────────────────────────────────────┘
 (F) スナックバー … 画面右下に積み重ねて表示（フローティング）
```

配置ルール:
- (C) ツールバーは一覧の **上**。左側に 3 ボタン、右側に「自動更新状況＋間隔セレクト」。
- 3 ボタンの並び順は **タスク実行（追加）→ 画面更新 → タスク中止** で固定。
- (D) テーブルの列順は **選択○・タスクID・追加日時・ステータス・実行時間・処理結果・処理件数・全体件数** で固定。
- (F) スナックバーは画面右下固定。複数同時表示は縦積み。

## 3. 表示項目（テーブル列）

| 列 | 元データ | 表示加工 |
|---|---|---|
| 選択(○) | （クライアント状態 selectedId） | ラジオ。行クリックでも選択可 |
| タスクID | `taskId` | 先頭 8 桁を表示、`title` に全 UUID |
| 追加日時 | `createdAt` | 日本語ロケールの日時 |
| ステータス | `status`/`statusLabel` | 色付きバッジ（[デザイン仕様](spec-design.md)参照） |
| 実行時間 | `executionMillis` | `12.3s` 形式。未開始は `—` |
| 処理結果 | `result` | そのまま |
| 処理件数 | `processedCount` | 数値＋進捗バー（processed/total）＋「成功x/失敗y」 |
| 全体件数 | `totalCount` | 数値 |

## 4. アクション定義（呼び出す API）

各アクションは **HATEOAS のリンク（rel）** を辿って API を呼ぶ。フロントはパス文字列を直書きしない。

| # | アクション | UI トリガ | 使用する rel（リンク元） | HTTP | 呼び出し後の挙動 |
|---|---|---|---|---|---|
| A1 | 初期表示 | 画面マウント | `tasks`（ルート `GET /api`） | GET | 一覧取得→描画。`create` リンクを保持 |
| A2 | タスク実行（追加） | 「タスク実行（追加）」ボタン | `create`（コレクション） | **POST** | 投入→当セッション集合に追加→`info` スナックバー→一覧再取得 |
| A3 | 画面更新 | 「画面更新」ボタン | `tasks`（保持済み href） | GET | 一覧を即時再取得 |
| A4 | タスク中止 | 「タスク中止」ボタン | 選択行の `cancel` | **POST** | 中止要求→`info` スナックバー→一覧再取得 |
| A5 | 自動更新 | タイマー（既定10秒） | `tasks` | GET | 一覧再取得（A3 と同じ）。間隔は (C) のセレクトで変更 |
| A6 | 行選択 | ラジオ/行クリック | （API 呼び出しなし） | — | `selectedId` を更新。中止ボタンの活性に影響 |
| A7 | 終了通知 | 自動更新の差分検知 | （API 呼び出しなし） | — | 当セッション投入タスクが終端化したら終了スナックバー（1回） |

> リンクの起点：
> - `create` は **A1/A3/A5 のコレクション取得時**に `_links.create` から取得・更新。
> - `cancel` は **各行（タスク）**の `_links.cancel`（未終端時のみ存在）。

## 5. 自動更新（ポーリング）仕様
- 画面を開いている間、**既定 10 秒間隔**で `tasks` を再取得（A5）。
- 間隔は **5 / 10 / 20 / 30 秒** から選択（(C) の Headless UI セレクト）。既定は 10 秒。
- 取得中は「最終更新」横にインジケータ（●）を表示。
- 取得失敗時は (B) エラーバナーを表示（バックエンド未起動の案内付き）。

## 6. スナックバー仕様
- 種類（トーン）: `success`(正常終了) / `warning`(警告終了) / `error`(異常終了) / `info`(中止終了・投入・中止要求)。
- **投入時**（A2）: 「タスク {id8} をキューに追加しました」(info)。
- **中止要求時**（A4）: 「タスク {id8} に中止を要求しました」(info)。
- **終了通知時**（A7）: 「タスク {id8} が『{ステータスラベル}』で終了しました（成功 x / 失敗 y）」。
  - 対象は **当セッションで投入したタスク**が終端に達したときのみ。各タスク 1 回だけ。
- 自動消滅（約 5.5 秒）。手動で × でも閉じられる。複数は右下に縦積み。

## 7. ボタンの活性条件

| ボタン | 活性条件 | 非活性時 |
|---|---|---|
| タスク実行（追加） | 常時（投入中は「追加中…」表示で一時非活性） | 投入処理中 |
| 画面更新 | 常時（取得中は「更新中…」で一時非活性） | 取得中 |
| タスク中止 | **未終端タスクを選択中**（選択行に `cancel` リンクがある） | 未選択 or 終端選択 or 中止処理中 |

## 8. 画面状態のパターン
- **空**: タスク 0 件 → テーブルに「タスクはまだありません…」を表示。
- **進行中あり**: `QUEUED`/`STARTED` 行が存在 → 進捗バーが伸び、実行時間が増加。
- **終端のみ**: すべて終端 → 中止ボタンは選択しても非活性。
- **通信エラー**: (B) バナー表示、一覧は直近成功時の内容を保持。

各コンポーネントの状態と見た目の詳細は **[デザイン仕様書](spec-design.md)** を参照。
